## CorePrinciples
1. **开发第一步必须是中间件部署**
   - 所有业务代码之前，先确保依赖环境可用
   - 用 Docker 快速启动，不追求复杂配置

2. **环境配置尽量简单**
   - 本地一套环境即可
   - Docker 仅用于中间件
   - 不引入复杂部署、CI/CD、多环境切换

3. **以“尽快开发、尽快上线”为最高优先级**
   - 优先打通最小业务闭环
   - 非核心能力可省略或延后

4. **测试文档必须可操作**
   - 每一个测试都必须写清楚“如何做”
   - 新手照文档即可复现结果
